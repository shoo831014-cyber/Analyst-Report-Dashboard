# AGENTS.md

## 목적
- 이 저장소는 `FnGuide` 리포트 수집 결과를 일자별로 집계해 대시보드로 보여주는 FastAPI 애플리케이션이다.

## 현재 작업 기준
- 기준 작업일: `2026-03-09`
- 서버 기본 주소: `http://127.0.0.1:8000`
- 대시보드: `http://127.0.0.1:8000/dashboard`
- 헬스 체크: `http://127.0.0.1:8000/health`

## 운영 규칙
- 같은 종목에서 같은 증권사의 리포트는 `1건`으로 본다.
- 한글/영문 중복 리포트도 위 규칙에 따라 같은 증권사면 `1건` 처리한다.
- `IPO` 성격 리포트는 집계에서 제외한다.
  - 제목에 `IPO` 포함
  - 종목코드가 `6자리 숫자`가 아닌 경우 포함
- Spotlight와 섹터 카드의 요약은 FnGuide 목록의 파란 bullet summary를 사용한다.
- 여러 리포트가 있으면 summary 문장을 모두 합치고, 문장 단위로 중복 제거한다.
- Spotlight 카드에서는 `애널리스트 n명` 숫자 표시는 노출하지 않는다.
- Spotlight 카드 하단 `제공처 n곳`에는 제공처/애널리스트 hover 정보가 연결된다.

## 섹터 통합 규칙
- `에너지`, `유틸리티`는 `에너지/유틸리티`로 통합한다.
- `자유소비재`, `필수소비재`는 `소비재`로 통합한다.
- `의료`, `헬스케어`, `바이오`는 `헬스케어/바이오`로 통합한다.
- `리브스메드`, `알지노믹스`는 `헬스케어/바이오`로 강제 매핑한다.

## 섹터 규칙
- 종목별 섹터는 첨부 엑셀 `d:\backup01\Desktop\종목 구분_대분류10.xlsx`의 `E열(대분류)`을 우선 사용한다.
- 엑셀 매핑이 없으면 기존 보조 규칙으로 처리한다.

## 다크 모드
- 헤더에 테마 토글이 있다.
- 선택한 테마는 브라우저 `localStorage`의 `dashboardTheme`에 저장한다.
- 저장값이 없으면 시스템 `prefers-color-scheme`를 따른다.

## 수동 업데이트
- 헤더의 `수동 업데이트` 버튼은 클릭 시 서울 기준 오늘 날짜로 다음 작업을 수행한다.
  - `ingest`
  - `publish`
  - `export`
- 버튼 클릭 시 진행중 상태를 표시하고, 완료 후 마지막 업데이트 시각을 헤더에 표시한다.

## EXE 실행기
- Windows 실행 파일은 `dist/AnalystReportDashboardServer.exe`다.
- 소스 엔트리포인트는 `server_launcher.py`다.
- 번들 실행 시 템플릿/정적파일 경로는 `app/runtime_paths.py`를 통해 해석한다.
- 재빌드 명령:
  - `.\.venv\Scripts\pyinstaller.exe pyinstaller_server.spec --noconfirm --clean`

## 구현 시 주의
- 대시보드 화면은 스냅샷을 읽는다. 로직 변경 후 서버 재시작만으로는 기존 날짜 데이터가 바뀌지 않으므로 재발행이 필요할 수 있다.
- 스냅샷 `created_at`은 UTC naive로 저장되며, 표시 시 서울시각으로 변환한다.
- 현재 Spotlight hover 정보는 브라우저 기본 `title` 툴팁 방식이다.
- 테스트 실행은 프로젝트 가상환경 기준으로 한다.
  - `.\.venv\Scripts\python.exe -m pytest`
